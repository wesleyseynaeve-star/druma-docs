## Mi az a TransFollow?

A TransFollow egy európai eCMR platform, amely digitális CMR fuvarokmányokat generál és kezeli a háromfeles aláírási folyamatot. A Druma a TransFollow API-ját integrálja, így az eCMR-ek automatikusan generálódnak a Drumából és az aláírásokat visszaszinkronizálják.

## TransFollow fiók létrehozása

Ha nincs TransFollow fiókod, látogasd meg a [transfollow.com](https://www.transfollow.org) weboldalt és regisztrálj. Szükséged lesz az API hitelesítési adatokra a Druma konfigurálásához.

## Konfigurálás a Drumában

<Steps>
  <Step title="Lépj a Beállítások → Integrációk → TransFollow menüpontra">
    Csak az Admin és Vállalati admin szerepkörök férnek hozzá.
  </Step>
  <Step title="Add meg az API hitelesítési adatokat">
    - **API kulcs** — a TransFollow API kulcsod
    - **Szállítói azonosító** — a TransFollow rendszerbeli vállalatod azonosítója
  </Step>
  <Step title="Teszteld a kapcsolatot">
    Kattints a **Kapcsolat tesztelése** gombra. A Druma teszteli az API elérhetőségét.
  </Step>
  <Step title="Mentés">
    Ha a teszt sikeres, mentsd el a konfigurációt. Az eCMR-ek mostantól automatikusan generálódnak.
  </Step>
</Steps>

## Automatikus vs. kézi eCMR

**Automatikus:** Az eCMR automatikusan generálódik, amikor egy rendelés **Kiosztva** státuszra kerül.

**Kézi:** Ha az automatikus generálás le van tiltva egy adott rendelésre, manuálisan is generálhatod az eCMR-t a rendelés részletei panelből.

## Hibaelhárítás

| Hiba | Ok | Megoldás |
|---|---|---|
| "API hitelesítés sikertelen" | Helytelen API kulcs | Ellenőrizd az API kulcsot a TransFollow konzolban |
| "Érvénytelen szállítói azonosító" | Helytelen szállítói ID | Ellenőrizd a TransFollow fiókodban |
| "eCMR generálás sikertelen" | Hiányzó kötelező mezők | Ellenőrizd, hogy az ÁFA-szám, cím és rakomány adatok ki vannak töltve |
