A Druma Nyilvános API-ja **csak olvasható** REST-hozzáférést biztosít a rendeléseidhez — hasznos egy ügyfél-önkiszolgáló portálhoz, egy BI/jelentéskészítő eszközhöz, vagy a csapatod által épített belső eszközökhöz.

<Warning>
A Nyilvános API egy funkció-jelzővel vezérelt képesség, és alapértelmezetten nincs engedélyezve. Írj a **support@druma.io** címre hozzáférés igényléséhez — az API-kulcsokat jelenleg közvetlenül a Druma állítja elő; nincs önkiszolgáló kulcsgeneráló képernyő az alkalmazásban.
</Warning>

## Hitelesítés

Minden API-kérésnek tartalmaznia kell az API-kulcsodat az `X-API-Key` fejlécben:

```
X-API-Key: A_TE_API_KULCSOD
```

Az érvényes kulcs nélküli kérések `401 Unauthorized` választ adnak. A Bearer-token hitelesítés (`Authorization: Bearer ...`) **nem** támogatott, és szintén 401-et ad vissza.

## Elérhető végpontok

Az API csak olvasható — csak `GET` kérések fogadhatók el. Minden válasz JSON formátumú.

| Metódus | Végpont | Leírás |
|---|---|---|
| `GET` | `/orders` | Rendelések listázása (lapozva, opcionális `status` szűrővel) |
| `GET` | `/orders/{id}` | Rendelés részletei |
| `GET` | `/orders/{id}/location` | A teherautó jelenlegi pozíciója és a legutóbbi ETA |
| `GET` | `/orders/{id}/documents` | Dokumentumlista, időben korlátozott aláírt letöltési URL-ekkel |

Bármely más erőforrás, vagy bármely nem-GET metódus `404`/`405` hibát ad vissza.

## Sebességkorlátok

- **100 kérés percenként** API-kulcsonként
- Ha meghaladod a korlátot, egy `429 Too Many Requests` választ kapsz egy `Retry-After` fejléccel, amely megmondja, hány másodpercet kell várnod
- Nincs más sebességkorlát-fejléc

Magasabb korlátokért írj a support@druma.io címre a felhasználási esetedet leírva.
