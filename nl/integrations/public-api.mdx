De openbare API van Druma geeft u **alleen-lezen** REST-toegang tot uw orders — nuttig voor een zelfbedieningsportaal voor klanten, een BI-/rapportagetool, of interne tooling die uw team heeft gebouwd.

<Warning>
De openbare API is een functie die niet standaard is ingeschakeld. Neem contact op met **support@druma.io** om toegang aan te vragen — API-sleutels worden momenteel rechtstreeks door Druma uitgegeven; er is geen self-service sleutelgeneratiescherm in de app.
</Warning>

## Authenticatie

Elk API-verzoek moet uw API-sleutel bevatten in de header `X-API-Key`:

```
X-API-Key: UW_API_SLEUTEL
```

Verzoeken zonder geldige sleutel krijgen een `401 Unauthorized`-respons. Bearer-token-authenticatie (`Authorization: Bearer ...`) wordt **niet** ondersteund en geeft ook een 401.

## Beschikbare endpoints

De API is alleen-lezen — alleen `GET`-verzoeken worden geaccepteerd. Alle responses zijn JSON.

| Methode | Endpoint | Beschrijving |
|---|---|---|
| `GET` | `/orders` | Orders opsommen (gepagineerd, optioneel `status`-filter) |
| `GET` | `/orders/{id}` | Orderdetail |
| `GET` | `/orders/{id}/location` | Huidige truckpositie en laatste ETA |
| `GET` | `/orders/{id}/documents` | Documentenlijst met tijdgebonden ondertekende downloadlinks |

Elke andere resource, of elke niet-GET-methode, geeft een `404`/`405`-fout.

## Snelheidslimieten

- **100 verzoeken per minuut** per API-sleutel
- Overschrijdt u de limiet, dan krijgt u een `429 Too Many Requests`-respons met een `Retry-After`-header die aangeeft hoeveel seconden u moet wachten
- Er worden geen andere snelheidslimiet-headers teruggegeven

Voor hogere limieten, neem contact op met support@druma.io met uw use case.
