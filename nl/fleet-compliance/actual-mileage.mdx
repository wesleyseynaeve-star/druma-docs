## Overzicht

Druma registreert de odometerwaarde bij het begin en einde van elke rit en berekent hoeveel kilometer de truck daadwerkelijk heeft gereden — los van de HERE-geplande routeafstand.

## Hoe het werkt

Wanneer een order overschakelt naar **Onderweg** registreert Druma `odometer_start_km` vanuit de telematicafeed van de truck. Wanneer de order overschakelt naar **Afgeleverd** registreert het `odometer_end_km`.

De werkelijke afstand is `odometer_end_km - odometer_start_km`.

## Ondersteunde telematica-providers

| Provider | Odometergegevensbron |
|---|---|
| **Geotab** | Volledige odometerfeed via MyGeotab SDK |
| **Webfleet** | WEBFLEET.connect API-odometerveld |
| **VDO** | Wagenparktelematica via provider API |
| **Frotcom** | Via provider API (beta) |
| **Webeye / Eurowag** | Via provider API (beta) |
| **rFMS (Scania, Volvo, DAF, MAN, Mercedes-Benz, IVECO, Renault)** | Totale voertuigafstand via rFMS standaard-adapter |

Alle providers worden verwerkt via Druma's `position-normaliser` Edge Function, die per positiepuls één rij per truck bijhoudt (GDPR single-row overwrite patroon).

## Gebruik van werkelijke vs. geplande afstand

| Gebruik | Welke afstand |
|---|---|
| Routeplanning en ETA | HERE-geplande afstand |
| CO2-berekening | HERE-geplande afstand (conform GLEC) |
| Brandstofkostenanalyse | Werkelijke afstand |
| Afwijkingsdetectie | Vergelijking van beide |
| Chauffeursprestatierapportage | Werkelijke afstand |

## Afwijkingen bekijken

Ga naar **Rapporten → Rijafwijkingsrapport** om orders te zien waarbij de werkelijke afstand significant afweek van de geplande route.
