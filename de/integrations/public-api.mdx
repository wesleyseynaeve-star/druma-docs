Die Druma Public API gibt Ihnen **schreibgeschützten** REST-Zugriff auf Ihre Aufträge — nützlich für ein Kunden-Selbstbedienungsportal, ein BI-/Berichtstool, oder interne Werkzeuge, die Ihr Team gebaut hat.

<Warning>
Die Public API ist eine per Feature-Flag gesteuerte Funktion und standardmäßig nicht aktiviert. Kontaktieren Sie **support@druma.io**, um Zugang zu beantragen — API-Schlüssel werden derzeit direkt von Druma bereitgestellt; es gibt keinen Selbstbedienungsbildschirm zur Schlüsselerzeugung in der App.
</Warning>

## Authentifizierung

Jede API-Anfrage muss Ihren API-Schlüssel im Header `X-API-Key` enthalten:

```
X-API-Key: IHR_API_SCHLÜSSEL
```

Anfragen ohne gültigen Schlüssel erhalten eine `401 Unauthorized`-Antwort. Bearer-Token-Authentifizierung (`Authorization: Bearer ...`) wird **nicht** unterstützt und gibt ebenfalls 401 zurück.

## Verfügbare Endpunkte

Die API ist schreibgeschützt — nur `GET`-Anfragen werden akzeptiert. Alle Antworten sind JSON.

| Methode | Endpunkt | Beschreibung |
|---|---|---|
| `GET` | `/orders` | Aufträge auflisten (paginiert, optionaler `status`-Filter) |
| `GET` | `/orders/{id}` | Auftragsdetails |
| `GET` | `/orders/{id}/location` | Aktuelle LKW-Position und neueste ETA |
| `GET` | `/orders/{id}/documents` | Dokumentenliste mit zeitlich begrenzten, signierten Download-URLs |

Jede andere Ressource oder jede Nicht-GET-Methode gibt einen `404`-/`405`-Fehler zurück.

## Ratenlimits

- **100 Anfragen pro Minute** pro API-Schlüssel
- Überschreiten Sie das Limit, erhalten Sie eine `429 Too Many Requests`-Antwort mit einem `Retry-After`-Header, der angibt, wie viele Sekunden zu warten sind
- Es werden keine weiteren Ratenlimit-Header zurückgegeben

Für höhere Limits kontaktieren Sie support@druma.io mit Ihrem Anwendungsfall.
