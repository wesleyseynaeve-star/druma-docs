Każda integracja, z której korzysta Twoja firma, jest konfigurowana w jednym miejscu: **Ustawienia → Integracje**. Nie potrzebujesz programisty — wszystkie konfiguracje są formularzowe i zajmują mniej niż pięć minut każda.

## 1. Dostawca eCMR

Druma wydaje elektroniczne listy przewozowe **we własnym zakresie, przy użyciu natywnego dostawcy Druma** — żadne zewnętrzne konto, klucz API ani konfiguracja nie są wymagane. eCMR jest dostępne od razu dla zleceń transgranicznych, a Druma pieczętuje każdy ukończony dokument cyfrowym podpisem PAdES.

Karta dostawcy eCMR oferuje tylko dwa stany: **Natywny** i **Wyłączony**. TransFollow nie jest opcją do wyboru — istnieje wyłącznie jako zapasowa ścieżka dla niewielkiej liczby firm z wcześniejszą konfiguracją (patrz notatka niżej).

<Steps>
  <Step title="Otwórz kartę dostawcy eCMR">
    Przejdź do **Ustawienia → Integracje** i otwórz kartę **Dostawca eCMR**. Domyślnie wybrany jest **Natywny**.
  </Step>
  <Step title="Przetestuj połączenie">
    Kliknij **Testuj połączenie**. Druma weryfikuje, czy certyfikat podpisu jest skonfigurowany i gotowy.
  </Step>
  <Step title="Zapisz">
    Kliknij **Zapisz**. Tworzenie eCMR jest aktywne dla zleceń transgranicznych.
  </Step>
</Steps>

<Note>
Większość firm nigdy nie musi tego zmieniać — Natywny to jedyna realna opcja. Jeśli Twoja firma ma wcześniejszą konfigurację TransFollow, nadal działa jak wcześniej i jest tu pokazywana jako **Natywny**; przy następnym zapisaniu tej karty po cichu migruje do prawdziwego natywnego dostawcy, bez niczego do przełączenia czy skonfigurowania. Zobacz [Dostawcy eCMR](/pl/ecmr/ecmr-providers) po szczegóły.
</Note>

## 2. HERE Maps i routing PTV

HERE Maps obsługuje autouzupełnianie adresów przy tworzeniu zleceń. Obliczanie trasy zoptymalizowanej dla ciężarówek, odległości, ETA i szacowania opłat drogowych obsługuje **PTV**, osobny dostawca routingu.

**Konfiguracja nie jest wymagana dla żadnego z nich.** Oba klucze API są zarządzane na poziomie platformy przez Druma. Nie ma nic do skonfigurowania i nie będziesz oddzielnie rozliczany za użycie HERE ani PTV — oba są wliczone w subskrypcję.

<Note>
Jeśli autouzupełnianie adresów lub routing przestanie działać, jest to problem na poziomie platformy. Skontaktuj się z pomocą techniczną Druma zamiast próbować samodzielnie rekonfigurować HERE lub PTV.
</Note>

## 3. Import kart paliwowych

Druma nie łączy się bezpośrednio z API dostawców kart paliwowych. Zamiast tego pobierasz eksport CSV z portalu internetowego swojego dostawcy i przesyłasz go do Druma. To automatycznie łączy koszty paliwa z pojazdami i zleceniami.

**Obsługiwani dostawcy:** DKV, UTA, AS24, Shell, TotalEnergies, BP, Circle K, Eurowag, WEG, MOL, E100.

<Steps>
  <Step title="Eksportuj z portalu dostawcy">
    Zaloguj się do portalu internetowego dostawcy karty paliwowej i wyeksportuj transakcje jako CSV. Większość portali oferuje to w sekcji **Raporty → Eksport transakcji**.
  </Step>
  <Step title="Otwórz Import kart paliwowych">
    Przejdź do **Flota → Paliwo** i otwórz okno importu.
  </Step>
  <Step title="Wybierz dostawcę">
    Wybierz dostawcę z listy rozwijanej, aby Druma wiedziała, jakiego formatu kolumn CSV oczekiwać.
  </Step>
  <Step title="Prześlij CSV">
    Kliknij **Prześlij CSV**, wybierz plik i kliknij **Importuj**. Druma dopasuje transakcje do pojazdów według numeru rejestracyjnego.
  </Step>
</Steps>

<Note>
Jeśli numer rejestracyjny transakcji nie pasuje do pojazdu w Twojej flocie, Druma wylistuje ją jako **niedopasowaną**. Możesz ją ręcznie przypisać lub zostawić. Regularnie sprawdzaj listę niedopasowanych — często wykrywa oszustwa kartą paliwową lub nieprawidłowe wpisy numerów rejestracyjnych.
</Note>

## 4. Księgowość SAGA / WinMENTOR

Druma eksportuje dane faktur i zleceń w formatach zgodnych z SAGA i WinMENTOR, dwoma najpopularniejszymi rumuńskimi pakietami księgowymi.

### SAGA Classic i WinMENTOR Classic

Klucz API nie jest potrzebny. SAGA eksportuje jako pliki DBF (`CLI_`, `ART_`, `IE_`, `INT_`, `NC_.xml`); WinMENTOR Classic eksportuje jako pliki `.txt` rozdzielane znakiem `|`. Oba formaty są generowane automatycznie raz w miesiącu i wysyłane e-mailem na skonfigurowany adres księgowego jako ZIP (Resend, podpisany link ważny 7 dni) — dzień eksportu skonfigurujesz w **Ustawienia → Automatyzacje i funkcje → Konfiguracja eksportu SAGA**. Nie ma ręcznej listy "Historia eksportów" — eksport każdego miesiąca istnieje wyłącznie jako wysłany e-mailem ZIP.

<Note>
Obecnie nie ma bezpośredniej integracji REST API dla WinMENTOR Enterprise ani żadnego innego pakietu księgowego — SAGA i WinMENTOR Classic są jedynymi dwoma obsługiwanymi formatami, oba oparte na plikach.
</Note>

## 5. SmartBill

SmartBill to rumuńska platforma fakturowania w chmurze. Po aktywacji tej integracji Druma automatycznie przesyła fakturę do SmartBill w momencie jej **wysłania** (nie w momencie utworzenia ani oznaczenia jako potwierdzona) — bez ręcznego ponownego wprowadzania danych.

<Steps>
  <Step title="Pobierz dane uwierzytelniające SmartBill">
    W koncie SmartBill znajdź swój **E-mail** konta, **Token API**, **CIF** (numer rejestracji podatkowej) i **Serię faktur** — wszystkie cztery są wymagane.
  </Step>
  <Step title="Otwórz kartę SmartBill w Druma">
    Przejdź do **Ustawienia → Integracje → SmartBill**.
  </Step>
  <Step title="Wprowadź dane uwierzytelniające">
    Wpisz E-mail, Token API, CIF i Serię faktur. SmartBill używa CIF do kierowania faktur do właściwego konta firmowego.
  </Step>
  <Step title="Zapisz">
    Kliknij **Zapisz**. Od tej chwili wysłanie faktury w Druma przesyła ją również do SmartBill.
  </Step>
</Steps>

<Note>
SmartBill nadaje własny numer faktury przy przesłaniu — nie ponownie wykorzystuje numeru faktury Druma. Status jest śledzony po prostu jako przesłano lub błąd; Druma nie śledzi stanu SmartBill Wersja robocza/Wystawiona/Opłacona i nie ma webhooka utrzymującego synchronizację obu systemów po przesłaniu (nie są też tworzone rekordy klienta ani produktu w SmartBill). Jeśli Twoja subskrypcja SmartBill wygaśnie lub Twój klucz API zostanie zregenerowany, przesyłanie zacznie zawodzić — okresowo sprawdzaj **Ustawienia → Integracje**, aby potwierdzić, że status SmartBill jest zielony.
</Note>

## 6. Fakturowanie elektroniczne Peppol (Belgia)

Peppol to ogólnoeuropejska sieć e-fakturowania. Belgijskie firmy (zarejestrowane w KBO) są zobowiązane do wysyłania faktur do innych belgijskich firm przez Peppol — jest to obowiązkowe od stycznia 2026.

**Wymagane dla:** belgijskich firm wysyłających faktury B2B do innych belgijskich firm.

<Steps>
  <Step title="Znajdź kartę Peppol E-Invoicing">
    Przejdź do **Ustawienia → Integracje** i kliknij kartę **Peppol E-Invoicing**.
  </Step>
  <Step title="Wprowadź dane rejestracyjne">
    Wpisz swój **numer KBO** (belgijski numer rejestracji firmy) i **numer VAT**.
  </Step>
  <Step title="Zarejestruj się w sieci Peppol">
    Kliknij **Zarejestruj się w sieci Peppol**. Druma przesyła Twoją rejestrację do punktu dostępu e-invoice.be. Zazwyczaj kończy się to w ciągu kilku minut.
  </Step>
  <Step title="Potwierdź aktywację">
    Po aktywacji karta pokazuje status **Połączono**. Routing Peppol jest teraz automatyczny dla uprawnionych belgijskich klientów — nie są potrzebne dodatkowe kroki przy wysyłaniu faktur do belgijskich odbiorców.
  </Step>
</Steps>

<Note>
Zobacz [Fakturowanie Peppol](/pl/integrations/peppol-invoicing), aby uzyskać pełne szczegóły dotyczące działania routingu, tego co się dzieje, gdy odbiorca nie jest w sieci Peppol, i jak obsługiwać odrzucenia.
</Note>

## 7. Skrzynka e-mail dla faktur dostawców

Daje Twojej firmie dedykowany adres e-mail do odbierania faktur od dostawców — od dostawców kart paliwowych, warsztatów, operatorów opłat drogowych czy dowolnego innego dostawcy. Otrzymane faktury są klasyfikowane przez AI i kierowane do kolejki do przeglądu, więc nic nie umyka.

<Steps>
  <Step title="Przejdź do Ustawień firmy">
    Przejdź do **Ustawienia → Firma** i przewiń do sekcji **E-mail faktur dostawców**.
  </Step>
  <Step title="Włącz skrzynkę">
    Włącz **Włącz skrzynkę faktur dostawców**. Druma generuje dedykowany przychodzący adres e-mail dla Twojej firmy.
  </Step>
  <Step title="Dodaj adresy nadawców do białej listy">
    W polu **Biała lista nadawców** dodaj adresy e-mail lub domeny, od których oczekujesz faktur (np. `invoices@dkv.com` lub `@uta.eu`). E-maile od nadawców spoza białej listy są odrzucane i nigdy nie trafiają do kolejki.
  </Step>
  <Step title="Udostępnij adres dostawcom">
    Skopiuj wygenerowany adres e-mail i podaj go każdemu dostawcy jako adres do przesyłania faktur dla Twojej firmy.
  </Step>
</Steps>

Otrzymane faktury pojawiają się w **Finanse → Spedycja → Kolejka faktur** do przeglądu, dopasowania i zatwierdzenia.

<Note>
Zobacz [Faktury dostawców](/pl/invoicing/supplier-invoices), aby uzyskać pełne szczegóły dotyczące kolejki przeglądu, logiki automatycznego dopasowania i obsługi niedopasowanych faktur.
</Note>

## 8. Dostawcy telematyki, tachografu i chłodni

Druma łączy się z GPS pojazdów, tachografem (godziny jazdy) i dostawcami chłodni naczep bezpośrednio z **Ustawienia → Integracje**, dzięki czemu pozycje na żywo, dane godzin jazdy wg EU 561/2006 i odczyty temperatury ładunku napływają do Druma automatycznie zamiast wymagać ręcznego wprowadzania.

### GPS i tachograf (telematyka)

Otwórz kartę swojego dostawcy w **Ustawienia → Integracje** i wprowadź poniższe dane uwierzytelniające. Link "Skąd je wziąć?" na każdej karcie powtarza tę wskazówkę w aplikacji.

| Dostawca | Pola uwierzytelniające | Skąd je wziąć |
|---|---|---|
| **Scania / Volvo / DAF / MAN / Mercedes-Benz / IVECO / Renault (rFMS)** | Client ID, Client Secret, selektor OEM | Wydawane przez portal deweloperski lub flotowy producenta ciężarówek (np. `developer.scania.com` dla Scania). Scania dodatkowo wymaga jednostki pokładowej Communicator oraz subskrypcji Location lub Performance. Scania i Volvo są najbardziej sprawdzone; pozostałe marki standardu rFMS współdzielą to samo API i mają działać tak samo, ale wciąż są weryfikowane na żywych flotach. |
| **Webfleet** | Konto, Nazwa użytkownika, Hasło, Klucz API | Od użytkownika Webfleet z profilem "expert" i włączonym dostępem zewnętrznym. Sam klucz API wydaje wsparcie Webfleet lub program partnerski WEBFLEET.connect. |
| **Geotab** | Baza danych, Nazwa użytkownika, Hasło | Utwórz konto serwisowe (tylko odczyt) w MyGeotab → Administracja → Użytkownicy. "Baza danych" to nazwa w Twoim adresie URL `my.geotab.com/<baza>`. |
| **Samsara** | Token API, Base URL (opcjonalnie) | Wygeneruj token w panelu Samsara pod Ustawienia → Tokeny API, z zakresem "Read Vehicle Statistics". Zostaw Base URL puste dla flot UE; ustaw je, jeśli organizacja działa na shardzie US. Tylko pozycje GPS — Samsara nie dostarcza danych tachografu/godzin jazdy. |
| **Frotcom** | Nazwa użytkownika, Hasło, Provider | Twoje dane konta Frotcom, z dostępem API włączonym przez wsparcie Frotcom lub lokalnego resellera. Obecnie w fazie **beta** — połączenie działa, ale mapowanie pól jest wciąż weryfikowane na żywych kontach. |
| **Continental VDO** | Client ID, Secret, ID konta/firmy | Wymaga umowy partnerskiej z Continental — skontaktuj się z Continental przez formularz kontaktowy `fleet.vdo.com`. **Dostęp tylko dla partnerów** — nie ma publicznej samoobsługowej rejestracji. |
| **Webeye / Eurowag** | Nazwa użytkownika, Secret | Generowane w `selfcare.eurowag.com` → Telematics → Fleet management → System connections. Twoja firma musi najpierw zostać zatwierdzona jako partner przez Eurowag. **Dostęp tylko dla partnerów** — nie ma publicznej samoobsługowej rejestracji. |

<Note>
Scania, Webfleet, Geotab i Samsara to sprawdzone integracje. Frotcom jest w fazie beta — działa, ale mapowanie pól jest wciąż potwierdzane. Continental VDO i Webeye/Eurowag wymagają zatwierdzenia Twojej firmy jako partnera przez dostawcę, zanim połączenie zwróci dane na żywo; formularz uwierzytelniania nadal pozwala zapisać i przetestować, ale synchronizacja nie ruszy, dopóki ten dostęp partnerski nie będzie aktywny.
</Note>

### Chłodnia / temperatura naczepy

<Steps>
  <Step title="Otwórz kartę dostawcy chłodni">
    W **Ustawienia → Integracje** znajdź kartę dostawcy telematyki Twojej naczepy — **Mapon**, **Orbcomm**, **Thermo King TracKing** lub **Carrier Lynx Fleet** — i otwórz ją.
  </Step>
  <Step title="Wprowadź dane uwierzytelniające">
    Link "Skąd je wziąć?" na każdej karcie wyjaśnia, jak uzyskać dane uwierzytelniające dla tego dostawcy (patrz tabela niżej).
  </Step>
  <Step title="Przetestuj i zapisz">
    Kliknij **Testuj połączenie**, następnie **Zapisz**. Odczyty temperatury zaczynają się synchronizować automatycznie po potwierdzeniu połączenia.
  </Step>
</Steps>

| Dostawca | Pola uwierzytelniające | Skąd je wziąć |
|---|---|---|
| **Mapon** | Klucz API, Base URL (opcjonalnie) | Wygeneruj klucz API w platformie Mapon pod Ustawienia → API. W pełni publiczne API — działa od razu. |
| **Orbcomm** | Token dostępu lub Nazwa użytkownika + Hasło, Base URL (opcjonalnie) | Wydawane przez konto ORBCOMM Platform API. Jeśli masz tylko nazwę użytkownika/hasło, Druma automatycznie wymienia je na token. |
| **Thermo King TracKing** | Klucz API lub Nazwa użytkownika + Hasło, Base URL (opcjonalnie) | Poproś dealera Thermo King lub portal TracKing o klucz API. Brak publicznej samoobsługowej rejestracji — to integracja dostępna tylko dla partnerów. |
| **Carrier Lynx Fleet** | Client ID + Client Secret (lub starszy Klucz API), Base URL (opcjonalnie) | Otrzymywane przez Carrier Transicold lub dealera Carrier jako część zestawu narzędzi Lynx Fleet API. Brak publicznej samoobsługowej rejestracji. |

<Note>
Mapon jest w pełni publiczny i działa od razu. Orbcomm potrzebuje tokenu konta od ORBCOMM, zanim zwróci dane na żywo. Thermo King TracKing i Carrier Lynx Fleet są dostępne tylko dla partnerów — będziesz musiał ubiegać się o dostęp API u dostawcy, zanim Druma będzie mogła pobierać dane na żywo, mimo że formularz połączenia jest dostępny dla wszystkich czterech.
</Note>
