Publiczne API Druma daje **tylko-do-odczytu** dostęp REST do Twoich zleceń — przydatne dla portalu samoobsługowego klienta, narzędzia BI/raportowego lub wewnętrznych narzędzi zbudowanych przez Twój zespół.

<Warning>
Publiczne API to funkcja wymagająca aktywacji i domyślnie nie jest włączona. Skontaktuj się z **support@druma.io**, aby zażądać dostępu — klucze API są obecnie wydawane bezpośrednio przez Drumę; nie ma samoobsługowego ekranu generowania kluczy w aplikacji.
</Warning>

## Uwierzytelnianie

Wszystkie żądania API muszą zawierać Twój klucz API w nagłówku `X-API-Key`:

```
X-API-Key: TWÓJ_KLUCZ_API
```

Żądania bez ważnego klucza zwracają odpowiedź `401 Unauthorized`. Uwierzytelnianie tokenem Bearer (`Authorization: Bearer ...`) **nie** jest obsługiwane i również zwraca 401.

## Dostępne endpointy

API jest tylko do odczytu — akceptowane są wyłącznie żądania `GET`. Wszystkie odpowiedzi są w formacie JSON.

| Metoda | Endpoint | Opis |
|---|---|---|
| `GET` | `/orders` | Lista zleceń (paginowana, opcjonalny filtr `status`) |
| `GET` | `/orders/{id}` | Szczegóły zlecenia |
| `GET` | `/orders/{id}/location` | Bieżąca pozycja ciężarówki i najnowsze ETA |
| `GET` | `/orders/{id}/documents` | Lista dokumentów z podpisanymi URL-ami do pobrania, ograniczonymi czasowo |

Każdy inny zasób lub dowolna metoda inna niż GET zwraca błąd `404`/`405`.

## Limity zapytań

- **100 żądań na minutę** per klucz API
- Jeśli przekroczysz limit, otrzymasz odpowiedź `429 Too Many Requests` z nagłówkiem `Retry-After` informującym, ile sekund odczekać
- Nie są zwracane żadne inne nagłówki limitu zapytań

W celu uzyskania wyższych limitów skontaktuj się z support@druma.io, podając swój przypadek użycia.
