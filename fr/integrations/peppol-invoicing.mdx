## Vue d'ensemble

Peppol (Pan-European Public Procurement On-Line) est l'infrastructure de réseau standard pour la facturation électronique en Europe. Pour les entreprises belges, l'envoi de factures B2B via Peppol est obligatoire depuis janvier 2026.

---

## Qui doit utiliser Peppol ?

Les entreprises belges (enregistrées au KBO) qui :
- Envoient des factures à d'autres entreprises belges (B2B)
- Ont un numéro de TVA belge actif

---

## Configuration

Voir [Configuration des intégrations → Peppol](/fr/admin/integration-setup) pour les instructions de configuration.

---

## Comment l'acheminement Peppol fonctionne

Lorsque vous envoyez une facture à un client belge :

1. Druma vérifie si le client est enregistré sur le réseau Peppol (via son numéro de TVA).
2. Si le client est sur Peppol, la facture est envoyée électroniquement via le réseau Peppol.
3. Si le client n'est pas sur Peppol, la facture est envoyée par email normalement.
4. Druma enregistre le statut de livraison Peppol dans le panneau de détails de la facture.

---

## Statuts Peppol

| Statut | Signification |
|---|---|
| **Envoyée** | Facture transmise au réseau Peppol |
| **Livrée** | Destinataire a reçu la facture |
| **Rejetée** | Destinataire a rejeté la facture (voir détails) |
| **Erreur** | Problème technique lors de la transmission |

---

## Gestion des rejets

Si une facture est rejetée par le destinataire Peppol :

1. Ouvrez la facture dans Druma.
2. Lisez les détails du rejet dans la section **Statut Peppol**.
3. Corrigez le problème identifié (ex. : numéro de TVA incorrect, format invalide).
4. Envoyez à nouveau la facture.
