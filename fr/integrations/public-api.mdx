L'API publique de Druma vous donne un accès REST **en lecture seule** à vos commandes — utile pour un portail libre-service client, un outil BI/reporting, ou des outils internes que votre équipe a construits.

<Warning>
L'API publique est une fonctionnalité à activation contrôlée et n'est pas activée par défaut. Contactez **support@druma.io** pour demander l'accès — les clés API sont actuellement provisionnées directement par Druma ; il n'y a pas d'écran de génération de clé en libre-service dans l'application.
</Warning>

## Authentification

Toutes les requêtes API doivent inclure votre clé API dans l'en-tête `X-API-Key` :

```
X-API-Key: VOTRE_CLE_API
```

Les requêtes sans clé valide retournent une réponse `401 Unauthorized`. L'authentification par jeton Bearer (`Authorization: Bearer ...`) n'est **pas** prise en charge et retourne aussi un 401.

## Endpoints disponibles

L'API est en lecture seule — seules les requêtes `GET` sont acceptées. Toutes les réponses sont en JSON.

| Méthode | Endpoint | Description |
|---|---|---|
| `GET` | `/orders` | Lister les commandes (paginé, filtre `status` optionnel) |
| `GET` | `/orders/{id}` | Détail d'une commande |
| `GET` | `/orders/{id}/location` | Position actuelle du camion et dernière ETA |
| `GET` | `/orders/{id}/documents` | Liste des documents avec URL de téléchargement signées à durée limitée |

Toute autre ressource, ou toute méthode non-GET, retourne une erreur `404`/`405`.

## Limites de taux

- **100 requêtes par minute** par clé API
- Si vous dépassez la limite, vous recevez une réponse `429 Too Many Requests` avec un en-tête `Retry-After` indiquant combien de secondes attendre
- Aucun autre en-tête de limite de taux n'est retourné

Pour des limites plus élevées, contactez support@druma.io avec votre cas d'usage.
