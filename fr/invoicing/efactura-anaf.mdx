## Qu'est-ce que l'e-Factura

e-Factura est le système de facturation électronique obligatoire de la Roumanie, géré par l'ANAF (l'administration fiscale roumaine). Depuis 2024, toutes les transactions B2B entre entreprises roumaines immatriculées à la TVA doivent être déclarées à l'ANAF au format XML UBL — le format de facture électronique standard utilisé à travers l'Europe.

En clair : chaque facture que vous émettez à une entreprise roumaine avec un CIF (numéro d'immatriculation TVA) doit être envoyée à l'ANAF électroniquement, pas seulement à votre client. Druma gère cela automatiquement une fois la facture générée.

## Comment Druma gère l'e-Factura

Vous n'avez pas besoin de vous connecter au portail ANAF, de générer des fichiers XML, ni de faire quoi que ce soit manuellement. Voici ce qui se passe automatiquement :

1. Vous générez une facture pour un client roumain ayant un CIF sur sa fiche client
2. Druma crée le fichier XML UBL au format ANAF correct
3. Druma soumet le XML à l'ANAF via l'API SPV (Spațiul Privat Virtual)
4. L'ANAF retourne un ID de soumission et, après traitement, un statut d'acceptation ou de rejet
5. La fiche de facture dans Druma se met à jour pour montrer le statut e-Factura actuel

## Suivi du statut e-Factura

Chaque facture nécessitant une soumission e-Factura affiche un badge de statut sur la page de détail de la facture sous l'onglet **e-Factura** :

| Statut | Signification |
|---|---|
| **En attente** | En attente de soumission (soumise généralement dans les minutes suivant la génération de la facture) |
| **Soumise** | XML envoyé à l'ANAF — en attente de traitement par l'ANAF |
| **Acceptée** | L'ANAF a validé et accepté la facture |
| **Rejetée** | L'ANAF a trouvé une erreur — voir le code d'erreur pour les détails |

<Note>
Le système de l'ANAF connaît régulièrement des indisponibilités, particulièrement les matins de semaine et en fin de mois. Si votre facture affiche « Soumise » pendant plus de quelques heures, c'est généralement du côté de l'ANAF, pas une erreur dans Druma. La file de nouvelles tentatives gère cela automatiquement.
</Note>

## File de nouvelles tentatives automatiques

Si une soumission échoue parce que l'ANAF est indisponible, Druma réessaie automatiquement avec un délai exponentiel, jusqu'à 3 tentatives au total (la tentative d'origine plus 2 nouvelles tentatives) :

- Première nouvelle tentative : environ **1 heure** après l'échec (±25 % de variation aléatoire)
- Deuxième nouvelle tentative : environ **2 heures** après la première nouvelle tentative (±25 % de variation aléatoire)

Si la troisième tentative échoue aussi, la facture passe directement au statut **Échec** — il n'y a plus de nouvelle tentative automatique après cela. Vous pouvez alors déclencher une **Resoumission manuelle** depuis l'onglet e-Factura de la page de facture.

## Consulter le statut e-Factura

<Steps>
  <Step title="Ouvrez la facture">
    Accédez à **Facturation** et cliquez sur la facture à vérifier.
  </Step>
  <Step title="Cliquez sur l'onglet e-Factura">
    L'onglet e-Factura montre le statut actuel, l'horodatage de soumission, l'ID de soumission ANAF (une fois reçu), et tout message d'erreur.
  </Step>
  <Step title="Si rejetée, lisez le code d'erreur">
    Les messages de rejet proviennent de l'ANAF et incluent un code d'erreur et une description. Les plus courants sont listés ci-dessous.
  </Step>
</Steps>

## Erreurs de rejet courantes

| Erreur | Cause probable | Comment corriger |
|---|---|---|
| **CIF invalide** | Le numéro de TVA du client sur sa fiche est incorrect ou non immatriculé à la TVA | Accédez à **Clients** → modifiez le client → corrigez le champ CIF |
| **Format d'adresse invalide** | Le champ rue, ville ou comté ne correspond pas au format attendu par l'ANAF | Modifiez l'adresse du client — assurez-vous que le Comté est défini et utilise le nom de comté roumain correct |
| **Champ obligatoire manquant** | Un champ XML requis est vide | Vérifiez la facture pour des détails client manquants — généralement le CIF, l'adresse, ou le nom du client |
| **Facture en double** | L'ANAF a déjà une soumission avec le même numéro de facture | Cela signifie généralement qu'une soumission précédente a réussi mais que Druma n'a pas reçu la confirmation — vérifiez directement le portail SPV de l'ANAF |

## Resoumission manuelle

Si les nouvelles tentatives automatiques échouent ou que vous avez corrigé une erreur et voulez réessayer immédiatement :

Accédez à **Facturation** → ouvrez la facture → **onglet e-Factura** → cliquez sur **Resoumettre à l'ANAF**. Druma envoie à nouveau le XML immédiatement.

## Quand l'e-Factura ne s'applique PAS

La soumission e-Factura n'est requise que pour les clients roumains immatriculés à la TVA. Elle ne s'applique **pas** à :

- Les factures transfrontalières UE (clients basés hors de Roumanie)
- Les clients sans CIF roumain
- Les factures aux particuliers (hors entreprise)

Pour ces factures, Druma génère une facture PDF standard sans l'étape de soumission XML UBL. L'onglet e-Factura n'apparaîtra pas sur ces factures.

<Warning>
Assurez-vous que chaque client B2B roumain dans Druma a son CIF correct saisi sur sa fiche client. Sans CIF, Druma ne peut pas soumettre à e-Factura et vous pourriez être non conforme. Vous pouvez vérifier et mettre à jour les numéros CIF sous Clients → sélectionnez le client → onglet Détails.
</Warning>
