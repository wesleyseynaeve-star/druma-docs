## Vue d'ensemble

Druma peut importer des relevés bancaires dans les formats CAMT.053 et MT940 — les deux formats standard pour les exports bancaires européens. Une fois importé, chaque transaction entrante apparaît dans la file de réconciliation.

---

## Formats pris en charge

| Format | Description |
|---|---|
| **CAMT.053** | Format XML standard ISO 20022, utilisé par la plupart des banques européennes |
| **MT940** | Format SWIFT hérité, toujours utilisé par de nombreuses banques |

---

## Importer un relevé bancaire

<Steps>
  <Step title="Exportez votre relevé depuis votre banque">
    Connectez-vous à votre banque en ligne et exportez les transactions de la période souhaitée en format CAMT.053 ou MT940.
  </Step>
  <Step title="Accédez à Facturation → Réconciliation → Importer un relevé">
    Cliquez sur **Importer un relevé bancaire**.
  </Step>
  <Step title="Sélectionnez votre fichier">
    Choisissez le fichier exporté depuis votre banque.
  </Step>
  <Step title="Vérifiez les transactions">
    Druma affiche un aperçu des transactions trouvées dans le fichier. Confirmez l'import.
  </Step>
</Steps>

---

## Ingestion par email

Si vous avez configuré la boîte de réception des emails de paiement (voir **Paramètres → Entreprise**), les emails de notification de paiement de votre banque sont automatiquement analysés et ajoutés à la file de réconciliation — sans import manuel.
