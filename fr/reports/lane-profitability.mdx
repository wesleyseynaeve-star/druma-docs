Beaucoup d'opérateurs connaissent leur revenu par ligne mais pas leur marge réelle une fois le carburant, les péages, et les coûts chauffeur comptés. Le rapport de rentabilité par ligne fait le calcul pour chaque combinaison de paire de pays que vous exploitez, afin que vous puissiez voir en un coup d'œil quelles lignes fonctionnent bien et lesquelles rongent votre profit.

## Ouvrir le rapport

Accédez à **Rapports → Rentabilité par ligne**.

Sélectionnez votre plage de dates en haut. Le rapport est par défaut sur le mois en cours. Pour une comparaison significative, vous voulez généralement au moins un mois complet de données, idéalement un trimestre.

## Ce que montre le rapport

Chaque ligne du tableau est une paire de pays origine-destination. Pour chaque ligne, vous voyez :

| Colonne | Signification |
|---|---|
| **Revenu** | Montant total facturé pour les commandes sur cette ligne durant la période |
| **Coût carburant** | Estimé : distance × consommation moyenne × prix du carburant |
| **Coût péage** | Données de péage PTV pour l'itinéraire |
| **Coût chauffeur** | Coût journalier × jours de trajet, basé sur le tarif par chauffeur que vous configurez |
| **Coûts totaux** | Somme carburant + péages + coût chauffeur |
| **Marge €** | Revenu moins coûts totaux |
| **Marge %** | Marge en pourcentage du revenu |
| **Trajets** | Nombre de commandes terminées sur cette ligne durant la période |
| **Prix moy./trajet** | Revenu moyen par commande |
| **Tendance** | Flèche indiquant si la marge % s'est améliorée ou dégradée par rapport à la période précédente |

Les lignes sous le seuil de marge cible sont surlignées en rouge. Ce seuil est actuellement fixé à **15 %** — il n'est pas encore configurable depuis les Paramètres.

## Comment fonctionnent les estimations de coût

<Note>
Les estimations de coût dans ce rapport sont des approximations basées sur vos paramètres configurés. Elles sont utiles pour identifier la performance relative entre les lignes, mais les coûts réels peuvent différer — particulièrement pour les coûts de péage sur les itinéraires avec des routes à péage variable.
</Note>

**Calcul du coût carburant :**
Distance (depuis le routage PTV) × consommation moyenne de la flotte (litres/100km, définie dans Paramètres → Flotte → Consommation par défaut) × votre prix du carburant (défini dans Paramètres → Flotte → Prix du carburant, ou importé depuis les données de cartes carburant si vous utilisez l'import de cartes carburant).

**Calcul du coût péage :**
Données de péage PTV pour l'itinéraire. Couvre la plupart des systèmes de péage autoroutier européens. Moins précis pour les itinéraires traversant des pays avec des systèmes de vignette complexes.

**Calcul du coût chauffeur :**
Nombre de jours de trajet × le tarif de coût journalier que vous avez défini pour chaque chauffeur dans son profil (Flotte → Chauffeurs → [chauffeur] → Tarif de coût journalier). Si aucun chauffeur n'est assigné, Druma utilise votre coût journalier chauffeur par défaut global.

Vous pouvez mettre à jour toutes ces entrées dans Paramètres pour améliorer la précision. Le rapport se recalcule immédiatement quand vous enregistrez de nouvelles valeurs.

## Identifier les lignes non rentables

Triez le tableau par **Marge %** croissante — les lignes les moins performantes remontent en haut.

Les lignes rouges sont sous votre marge cible. Raisons courantes :

- **Lignes longue distance** — semblent rentables en revenu mais ont des coûts carburant et chauffeur élevés
- **Lignes à péages lourds** — certains itinéraires à travers la France, l'Autriche, ou la Suisse ont des péages significativement plus élevés que les alternatives
- **Clients historiques sous-tarifés** — tarifs convenus il y a des années et jamais renégociés
- **Lignes à faible volume** — moins de trajets signifie que les coûts fixes pèsent plus lourd

## Explorer une ligne en détail

Cliquez sur n'importe quelle ligne pour voir les commandes individuelles qui composent les chiffres de cette ligne. Vous pouvez voir le revenu, le coût estimé, et la marge de chaque commande — et identifier des commandes spécifiques qui sont des anomalies (par exemple, une seule commande à un tarif bien plus bas que le reste).

Depuis la liste des commandes, vous pouvez cliquer pour accéder au détail complet de la commande. Utile pour enquêter sur pourquoi une ligne spécifique a sous-performé un mois donné.

## Filtrer le rapport

Utilisez les filtres en haut de la page :

- **Pays d'origine** — filtrez sur un ou plusieurs pays d'origine spécifiques
- **Pays de destination** — filtrez sur un ou plusieurs pays de destination spécifiques
- **Client** — voyez la rentabilité par ligne pour un seul client
- **Camion** — voyez quels camions exploitent quelles lignes le plus rentablement

## Exporter

Cliquez sur **Exporter → CSV** pour télécharger le rapport complet sous forme de tableur. Le CSV inclut toutes les colonnes et toutes les commandes de la vue explorée si vous avez appliqué des filtres.

Le CSV est le meilleur format pour une analyse plus poussée dans Excel — par exemple, construire vos propres tableaux croisés dynamiques par client ou par camion.

<Warning>
Ne prenez pas de décisions de tarification basées uniquement sur ce rapport sans considérer d'autres facteurs : les frets retour, les relations client stratégiques, et la variation saisonnière de volume affectent tous la valeur réelle d'une ligne.
</Warning>
