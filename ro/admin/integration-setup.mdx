Toate integrarile companiei tale se configureaza intr-un singur loc: **Setari -> Integrari**. Nu ai nevoie de un dezvoltator - fiecare configurare se face printr-un formular, in cateva minute.

---

## 1. Furnizor eCMR

Druma emite scrisori de transport electronice **intern, folosind furnizorul nativ Druma** - fara cont extern, cheie API sau configurare. eCMR este disponibil imediat pentru comenzile transfrontaliere, iar Druma sigileaza fiecare document finalizat cu o semnatura digitala PAdES.

Cardul are doar doua stari posibile: **Nativ** si **Dezactivat**. TransFollow nu mai poate fi selectat de nicio companie noua - exista doar ca ruta mostenita pentru companiile care aveau deja o configurare TransFollow.

<Steps>
  <Step title="Deschide cardul Furnizor eCMR">Furnizorul Nativ este selectat implicit.</Step>
  <Step title="Testeaza conexiunea">Druma verifica ca certificatul de semnare este configurat si pregatit.</Step>
  <Step title="Salveaza">eCMR devine activ pentru comenzile transfrontaliere.</Step>
</Steps>

<Note>
Daca o companie are o configurare TransFollow mostenita, aceasta continua sa functioneze si apare aici ca **Nativ** - migreaza automat la furnizorul nativ real la urmatoarea salvare, fara nimic de schimbat manual. Detalii in [Furnizori eCMR](/ro/ecmr/ecmr-providers).
</Note>

---

## 2. HERE Maps

Alimenteaza autocompletarea adreselor si calculul rutei optimizate pentru camioane (distanta, timp estimat, CO2). **Nu necesita configurare** - cheia API este gestionata la nivel de platforma si inclusa in abonament.

---

## 3. Import carduri de combustibil

Druma nu se conecteaza direct la API-urile furnizorilor - exporti CSV din portalul furnizorului si il incarci in Druma, iar tranzactiile sunt asociate automat cu vehiculele dupa numarul de inmatriculare.

**Furnizori acceptati:** DKV, UTA, AS24, Shell Card, SmartDiesel.

Mergi la **Setari -> Integrari -> Import Card Combustibil**, alege furnizorul din lista (pentru formatul corect de coloane CSV), incarca fisierul. Tranzactiile nepotrivite cu niciun vehicul apar ca **nepotrivite** - verifica-le periodic, deseori indica fraude sau plate gresite.

---

## 4. Contabilitate SAGA / WinMENTOR

Pentru SAGA Classic si WinMENTOR Classic: fara cheie API - generezi un export structurat (XML/CSV) din **Rapoarte -> Export Contabilitate** si il imporți cu wizardul standard al programului tau.

Pentru **WinMENTOR Enterprise** (API REST): introduci URL-ul endpoint-ului, utilizatorul si parola API furnizate de administratorul WinMENTOR, apoi testezi si salvezi.

---

## 5. SmartBill

Platforma romaneasca de facturare in cloud. Cu integrarea activa, facturile confirmate in Druma sunt trimise automat catre SmartBill.

Copiaza cheia API din **Cont -> API** in contul tau SmartBill, introdu cheia si CIF-ul companiei in cardul SmartBill din Druma, testeaza conexiunea si salveaza.

<Warning>
Daca abonamentul SmartBill expira sau cheia API este regenerata, integrarea se opreste silentios. Verifica periodic statusul (verde = activ).
</Warning>

---

## 6. Peppol e-Facturare (Belgia)

Reteaua paneuropeana de e-facturare - obligatorie din ianuarie 2026 pentru companiile belgiene (inregistrate KBO) care trimit facturi B2B catre alte companii belgiene.

Introdu numarul KBO si numarul de TVA in cardul **Peppol e-Facturare**, apoi apasa **Inregistreaza-te in Reteaua Peppol** (Druma trimite inregistrarea catre punctul de acces e-invoice.be). Odata activ, rutarea catre clientii belgieni eligibili este automata.

---

## 7. Inbox email facturi furnizori

Ofera companiei tale o adresa de email dedicata pentru facturile de la furnizori (carduri combustibil, service, taxe de drum). Facturile primite sunt clasificate automat prin AI si trimise intr-o coada de revizuire.

Activeaza optiunea in **Setari -> Companie -> Email Facturi Furnizori**, adauga adresele/domeniile expeditorilor asteptati in lista alba, apoi distribuie adresa generata furnizorilor tai. Facturile apar in **Finante -> Forwarding -> Coada Facturi**.

---

## 8. Telematica, tahograf si reefer

Druma se conecteaza direct la GPS-ul camioanelor, tahograful (ore de conducere EU 561/2006) si senzorii de temperatura ai remorcilor reefer, eliminand introducerea manuala.

**Telematica/tahograf** - deschide cardul furnizorului tau si introdu credentialele:

| Furnizor | Credentiale | Observatii |
|---|---|---|
| Scania / Volvo / DAF / MAN / Mercedes-Benz / IVECO / Renault (rFMS) | Client ID, Client Secret | Emise de portalul de dezvoltatori al producatorului. Scania si Volvo sunt cele mai consolidate; celelalte marci rFMS folosesc acelasi API dar sunt inca in validare pe flote live. |
| Webfleet | Cont, Utilizator, Parola, Cheie API | Necesita profil "expert" cu acces extern activat |
| Geotab | Baza de date, Utilizator, Parola | Cont de serviciu (View Only) creat in MyGeotab |
| Samsara | Token API | Doar pozitii GPS - nu ofera date de tahograf |
| Frotcom | Utilizator, Parola, Furnizor | In **beta** - functioneaza, dar maparea campurilor e in validare |
| Continental VDO | Client ID, Secret, ID cont | Necesita acord de partener Continental - fara auto-inregistrare publica |
| Webeye / Eurowag | Utilizator, Secret | Necesita aprobare de partener Eurowag - fara auto-inregistrare publica |

Automatizarea descarcarii tahografului functioneaza in acest moment doar pentru camioanele conectate prin rFMS (marcile OEM de mai sus); Webfleet TachoShare inca nu este implementat.

**Reefer** - Mapon (API public, functioneaza imediat), Orbcomm (token de cont necesar), Thermo King TracKing si Carrier Lynx Fleet (doar prin acord de partener cu furnizorul).
