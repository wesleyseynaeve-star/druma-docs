Setarile **Branding si Formate** controleaza cum apare compania ta pe documentele generate si cum se construiesc numerele de referinta pentru facturi si comenzi.

Aceste setari sunt specifice fiecarei companii. Daca administrezi mai multe companii in Druma, fiecare are propria configuratie de branding si format.

**Cine poate edita:** doar rolurile Admin si Company Admin.

## Accesarea setarilor

Mergi la **Setari → Branding si Formate**. Pagina are patru taburi: **Branding**, **Format Factura**, **Format Comanda** si **Formatare randuri**.

---

## Tabul Branding

Controleaza cele doua culori de brand folosite pe PDF-urile generate si sabloanele de email: **culoarea principala** (borduri, etichete, totaluri) si **culoarea accent** (antete de tabel, text de continut). Fiecare culoare are un camp cod hex plus un selector vizual, cu previzualizare live.

Apasa **Salveaza modificarile** sau **Reseteaza la implicit**.

<Note>
Logo-ul companiei NU se configureaza pe aceasta pagina — se incarca la **Setari → Companie**. Druma accepta fisiere PNG, JPEG, SVG sau WebP de pana la 2 MB; nu exista o cerinta minima de latime sau de fundal transparent impusa.
</Note>

---

## Tabul Format Factura

Controleaza patternul numarului de factura, titlul antetului, dobanda de intarziere, instructiunile de plata, termenii si conditiile, textul de subsol si sectiunile optionale afisate pe PDF-ul facturii.

### Token-uri de format

| Token | Ce insereaza |
|---|---|
| `{PREFIX}` | Prefixul definit de tine (ex: `INV`, `ORD`, propria abreviere) |
| `{YYYY}` | An pe 4 cifre (ex: `2026`) |
| `{YY}` | An pe 2 cifre (ex: `26`) |
| `{MM}` | Luna pe 2 cifre (ex: `04`) |
| `{DD}` | Ziua pe 2 cifre (ex: `07`) |
| `{YYMMDD}` | An, luna, zi combinate pe 2 cifre fiecare (ex: `260407`) |
| `{SEQ3}` | Numar secvential, completat cu zerouri la 3 cifre (ex: `042`) |
| `{SEQ4}` | Numar secvential, completat cu zerouri la 4 cifre (ex: `0042`) |
| `{SEQ5}` / `{SEQ6}` | Numar secvential, completat cu zerouri la 5 / 6 cifre |

Nu exista token `{NNN}`/`{NNNN}` — foloseste unul din token-urile `{SEQ3}`–`{SEQ6}` pentru o secventa cu zerouri. Poti combina token-urile liber cu separatori precum `-` sau `/`.

Secventa se reseteaza de fiecare data cand un token de data din pattern (`{YYMMDD}`, `{DD}`, `{MM}`, `{YY}` sau `{YYYY}`) isi schimba valoarea — practic, un pattern cu `{YYMMDD}` se reseteaza **zilnic**, nu anual. Un pattern care foloseste doar `{YYYY}` se reseteaza o data pe an, cum te-ai astepta.

**Implicit:** prefix `INV`, pattern `{PREFIX}-{YYMMDD}-{SEQ3}` → produce ceva de genul `INV-260407-042`.

| Sablon | Exemplu rezultat |
|---|---|
| `{PREFIX}-{YYYY}-{SEQ4}` | `INV-2026-0042` |
| `{PREFIX}/{YY}/{SEQ3}` | `INV/26/042` |
| `{YYYY}{MM}-{SEQ3}` | `202604-042` |

<Note>
Notele de credit folosesc propria serie de numerotare (prefix implicit `CN`) si propriul contor, dar nu au momentan un camp de pattern editabil pe aceasta pagina.
</Note>

Apasa **Salveaza modificarile**. Noul format se aplica urmatoarei facturi create dupa salvare — facturile existente pastreaza numarul de referinta cu care au fost emise.

## Tabul Format Comanda

Defineste patternul de numerotare pentru numerele de referinta ale comenzilor — identificatorul afisat pe planning board, in panoul de detaliu al comenzii si in comunicarile cu soferii sau clientii.

Acelasi sistem de token-uri ca la facturi. Introdu un **Prefix** si un **Pattern numar**; previzualizarea live se actualizeaza sub campuri.

**Implicit:** prefix `ORD`, pattern `{PREFIX}-{YYMMDD}-{SEQ3}` → produce ceva de genul `ORD-260407-042`.

**Exemplu:** un transportator din Belgia ar putea folosi prefixul `BE` cu patternul `{PREFIX}-{YYYY}-{SEQ4}` pentru referinte de tipul `BE-2026-0042`.

Apasa **Salveaza modificarile**. Noul format se aplica tuturor comenzilor create dupa salvare; comenzile existente isi pastreaza numarul original.

---

## Tabul Formatare randuri

Iti permite sa evidentiezi randurile din tabelele Druma pe baza unor conditii simple — de exemplu, colorezi un rand cu rosu cand marja scade sub un prag, sau cand este restant.

Fiecare regula are un nume, un camp de evaluat (marja %, este restant, status, zile de la creare), o conditie (mai mic decat, cel mult, mai mare decat, cel putin, egal, diferit), o valoare de comparat si o culoare de evidentiere. Apasa **Adauga regula** pentru a crea una si **Salveaza modificarile** pentru a aplica regulile.
